## 사용한  git 명령어 정리

***- git init*** : desktop 에 **wecode 폴더***의 모든 변경사항 추적/관리
***- git stats*** :어떤 파일이 변경되었는지, 어떤 파일이 추가되었는지 등의 상태호학인
***- git log*** : 프로젝트의 모든 커밋 내역을 본다
***- git add .*** : ```git add``` 라는 명령어를 통해 우리가 원하는 파일들을 staging area 로 추가해줄 수 있습니다. 
***-git commit -m "Commit message"*** : 커밋은 특정 시간의 코드 스냅샷의 형태로 해당 repository의 커밋 기록에 남게됩니다. `git add` 명령어를 사용하여 모든 파일을 staging area에 추가 해주었다면 이제 커밋을 남길 준비가 되었습니다.
***git remote add origin https://github.com/junchi211/wecode.git*** : Remote  할 주소 입력
***git push origin master*** : git hub 에 업로드
***git checkout -b feature/README**: feature/README branch를 만들면서 해당 branch로 이동
